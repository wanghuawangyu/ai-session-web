# v0.1.0 — AI Session Web

首个可用版本。用于管理和浏览本地 AI CLI 会话文件（Jcode、Codex、Continue）的 Web UI 工具。

> **发布日期**: 2026-07-19  
> **版本号**: 0.1.0  
> **发布类型**: Alpha（功能可用，持续开发中）

---

| 平台 | 文件 | 大小 |
|---|---|---|
| Windows (x86_64) | `ai-session-web.exe` | 2374 KB |

编译优化：LTO + strip + codegen-units=1（最小体积），单文件免安装。

## 功能清单

### 多来源会话管理

- 支持 **Jcode**（`.json` + `.journal.jsonl` 双文件合并）、**Codex**（`.jsonl`）、**Continue**（`.json`）三种 CLI 工具的会话文件
- CLI 目录自动识别：通过路径字符串中的关键字（`jcode` / `codex` / `continue`）推断来源类型
- 支持空格分隔或逗号分隔两种参数输入方式

### 会话扫描与解析

| 来源 | 文件格式 | 解析能力 |
|---|---|---|
| Jcode | `.json` + `.journal.jsonl` | 消息统计、custom_title、时间轴、cwd 覆盖、provider |
| Codex | `.jsonl` | 消息统计、时间戳、cwd、provider；容忍无效行 |
| Continue | `.json` | 消息统计、title、workspaceDirectory |

- Jcode 会话自动合并 `.json` 主文件与 `.journal.jsonl` 增量日志的消息数和时间轴
- 容忍损坏/无效的 JSON 行，不影响其他会话

### 智能排序输出

```
按 --cli-dirs 传入顺序 → 每个 CLI 类型分组
  ├── 主会话（有 custom_title） & 其关联临时会话（同 working_dir）
  │   ├── 主会话A ── 关联临时A1, 关联临时A2
  │   └── 主会话B ── 关联临时B1
  ├── 其他临时会话（不匹配任何主会话）
  └── 残缺会话（文件存在但解析失败）
```

- 主会话按 `effective_updated_at`（内容时间与文件 mtime 取大值）降序
- 每个主会话的关联临时会话紧随其后，缩进显示
- 残缺会话用删除线 + 灰色样式标记

### 会话查看

- 列表显示：名称 / Session ID / 消息统计（👤 🤖 💬）+ 提供商标签
- 项目信息：工作目录路径 + 创建时间（🕐）与更新时间（📝）
- 点击"查看"弹出模态框，显示格式化 JSON 内容
- Jcode 会话的 JSON 查看会合并 .journal.jsonl 中的消息

### 会话删除

- 每个会话行都有独立的删除按钮
- 删除时扫描会话目录，**删除所有文件名包含该 Session ID 的文件**（`.json`、`.jsonl`、`.bak`、`.journal.jsonl`）
- **主会话二次确认保护**：删除有 custom_title 的主会话时，需要两次确认（第一次警告→第二次确认"此操作不可恢复"）
- 非主会话单次确认即可删除

### 前端界面

- 纯原生 JavaScript + CSS（零前端框架依赖）
- 三列布局：名称/ID/消息 | 项目 | 操作
- 主会话行蓝色高亮背景：`#f0f4ff`
- 关联临时会话左缩进 40px + 浅灰背景
- 残缺会话：`opacity: 0.7` + 删除线
- 响应式适配（768px 以下列堆叠）
- 模态框：JSON 预览（暗色背景等宽字体） + 删除确认
- 支持点击 overlay / Escape 键关闭模态框

### 配置页面

- 访问 `/config` 查看当前运行时配置
- 显示：监听端口/地址、各 CLI 会话目录、日志级别、日志文件路径

### 日志系统

- 支持控制台输出 + 文件日志双通道
- 级别控制：`--log-level` 参数（trace / debug / info / warn / error）
- 每个扫描/删除操作记录 info 级日志

## 使用方式

```bash
# 使用默认目录（~/.jcode/sessions, ~/.codex/sessions, ~/.continue/sessions）
ai-session-web.exe

# 指定自定义目录（空格分隔）
ai-session-web.exe --cli-dirs "D:\data\.jcode\sessions" "D:\data\.codex\sessions"

# 指定自定义目录（逗号分隔）
ai-session-web.exe --cli-dirs "D:\data\.jcode\sessions,D:\data\.codex\sessions"

# 自定义端口
ai-session-web.exe --port 8080

# 开启文件日志
ai-session-web.exe --log ./app.log --log-level debug
```

启动后访问 http://127.0.0.1:8100

## API 接口

| 方法 | 路径 | 说明 |
|---|---|---|
| GET | `/api/sessions` | 获取所有会话（排序后分组结构） |
| DELETE | `/api/sessions/{source}/{session_id}` | 删除会话及其关联文件 |
| GET | `/api/sessions/{source}/{session_id}/json` | 获取会话原始 JSON（Jcode 含 journal 合并） |

```json
{
  "success": true,
  "data": [{

        { "session_id": "temp_1", "name": "" }

      "sectionType": "unlinked_temp",

      "sessions": []
    }, {
      "sectionType": "broken",
      "title": "残缺会话",
      "broken": []
    }]
  }]
}
```

## 已知限制

- 仅支持 Windows 平台（路径处理使用 `\`）
- 前端界面仅中文
- 不支持会话编辑/重命名
- 不支持搜索/过滤
- Continue 会话无时间轴字段
- 不支持热重载（需重启扫描新文件）

---

## 技术栈

- **后端**: Rust + Axum 0.8 + Askama 0.12 + Tokio + Tower/Tower-HTTP
- **前端**: 原生 JavaScript（ES2015+） + CSS3，无框架
- **存储**: 直接读取磁盘文件，无数据库
- **构建**: LTO + strip + codegen-units=1 优化
